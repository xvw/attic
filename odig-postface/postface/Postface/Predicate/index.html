Introduction

When manipulating higher order functions, it is quite common to use predicates to find a value in a structure or to filter it. That is, functions that for any value, return true or false.

Even if the predicates are only aliases, it is possible to simply compose them. Here is a simple example of a predicate on integers that returns true if the number is even, false otherwise.

# let is_even x = (x mod 2) = 0 ;;
val is_even : int -> bool = <fun>

There is no additional ceremony to describe a predicate than simply defining a one-argument function that returns a boolean.

API

Running predicates

A predicate can be executed. It is only a matter of giving it a value.

val run : 'a t -> 'a -> bool

run p x is equivalent to p x. It runs the predicate p on the value x. See examples

Predefined predicates

Some predefined predicates that do not depend on the input.

val tautology : 'a t

A tautology is a predicate that always return true no matter what the input. See examples

val contradiction : 'a t

A contradiction is a predicate that always return false no matter what the input. See examples

Acting on and composing predicates

val negate : 'a t -> 'a t

negate p negates the predicate p so run (negate p) x = not (run p x). See negate_examples for examples. See examples

val and_ : 'a t -> 'a t -> 'a t

and_ p1 p2 produces a predicate that apply p1 and p2 on one input so run (and_ p1 p2) x = (p1 x) && (p2 x). See examples

val or_ : 'a t -> 'a t -> 'a t

or_ p1 p2 produces a predicate that apply p1, and if p1 is not satisfied, it applies p2 on one input so run (or_ p1 p2) x = (p1 x) || (p2 x). See examples
