Note about Decidable and Divisible

Unlike Predicate and Equivalence, the implementations of Divisible and Decidable are less obvious. Indeed, one can consider a predicate to be the composition of two other predicates if both predicates are true (and equivalence is a more general case of predicate), considering the result of a comparison for products or sums is less natural.

Constructing a comparison that acts on products or sums takes advantage of the idempotent nature of the Ordering monoid. This implies that the order of projection in products (in pairs) or in sums (in Either) matters in the final result of the comparison. This makes these two implementations somewhat less useful than Predicate and Equivalence.

module Divisible : Preface.Specs.DIVISIBLE with type 'a t = 'a t

Examples of using Comparisons as a divisible. This allows the construction of more complex comparisons that act on product types.

module Decidable : Preface.Specs.DECIDABLE with type 'a t = 'a t

Examples of using Comparisons as a decidable. This allows the construction of comparisons that vary according to the branches of a sum type.

API

Helpers for dealing with Stdlib

Since OCaml's Stdlib deals with integer, it could be useful to be able to convert comparisons to returning integer.

val to_stdlib : 'a t -> 'a -> 'a -> int

to_stdlib c converts a Postface comparison function to an OCaml's Stdlib comparison function. See examples

val from_stdlib : ('a -> 'a -> int) -> 'a t

from_stdlib c converts an OCaml's Stdlib comparison function to a Postface comparison function. See examples

Running comparisons

A comparison can be executed. It is only a matter of giving it values.

val run : 'a t -> 'a -> 'a -> Ordering.t

run e x y is equivalent to e x y. It runs the comparison e on the values x and y. See examples
